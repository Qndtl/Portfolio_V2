Intro
인스타그램 클론 코딩하였다.
이 프로젝트는 Prisma1, React, GraphQL, Apollo Server를 사용하였다.
Login은 passport, jsonwebtoken을 사용하였다.
Prisma
Backend는 Prisma1을 사용하였다.
GraphQL-yoga를 사용하였고 파일을 업로드 할 때는 GraphQLServer에 내장되어있는 express를 사용하여 REST로 업로드 해주었다.
Log In
로그인은 passport-jwt, json web token을 사용하여 localStorage에 토큰을 저장하는 방식을 사용하였다.
로그인을 하기 위해서는 당연히 가입이 필요하고 가입후 로그인을 하면 가입을 했던 이메일로 loginSecret을 포함한 이메일이 온다.
이메일은 nodemailer, nodemailer-mailgun-transport를 사용하여 전송하였다.
Alert 대신 Toastify를 사용하여 알림을 준다.
Feed
로그인 성공 후 사용자들이 올린 게시물들이 보여진다.
게시물을 업로드 할 때 여러장의 사진을 올린 경우에는 useEffect를 이용해 사진들이 자동으로 슬라이드된다.
좋아요와 댓글의 경우는 fake state를 만들어 주어 즉시 변경이 적용되도록 하였다.
Search
검색을 하면 사용자와 게시물의 검색 결과가 나온다.
사용자 검색은 검색하기 위해 입력한 텍스트를 포함하는 username, firstname, lastname이 있는 사용자를 찾아주고 없는 경우 User Not Found를 보여준다.
게시물 검색은 검색하기 위해 입력한 텍스트를 포함하는 caption, location이 있는 게시물을 찾아주고 없는 경우 Post Not Found를 보여준다.
Upload
Prisma에는 Post의 id, File의 id, url 등이 저장이 되고 사진 자체는 AWS S3에 저장이 된다.
AWS S3에 사진을 업로드하기 위해 multer, multer-s3를 사용하였다.
GraphQL을 사용하지만 사진 업로드만 REST를 사용하였다.
업로드를 하고나면 다시 Feed 페이지로 이동한다.
Profile
프로필에는 Avatar, username, fullname, follower, following, 내가 작성한 게시물이 표시된다.
